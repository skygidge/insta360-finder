## Story #1 — TOP PICK

### Joseph B. Schulz
**Location:** Stanford, California, USA
**Product:** Insta360 X3
**Source:** Frontiers in Oncology, open-access, Vol. 15 (30 May 2025)
**Confidence:** Medium (source returns 403 — flag for human verification)

**Story summary:** Radiation oncologist at Stanford University School of Medicine who led a pilot study using the Insta360 X3 360-degree camera to photograph critical areas of the radiation oncology department — including reception, waiting rooms, consultation rooms, CT simulation suite, treatment planning room, and linear accelerator vaults. The team built an interactive panoramic virtual tour accessible via Stanford's MyHealth patient portal, allowing cancer patients scheduled for external beam radiation therapy to familiarise themselves with the treatment environment before their first appointment. A randomised controlled trial with 68 patients (33 control, 35 experimental) evaluated the tour's feasibility. Published in Frontiers in Oncology on 30 May 2025.

**Why Insta360 should feature this:** First Stanford-published clinical trial to use an Insta360 consumer camera as a clinical tool — demonstrating that a $400 action camera can help reduce anxiety in cancer patients. Opens an entirely new healthcare sector conversation for Insta360, complementing existing academic stories (Olga Kukharuk's PTSD therapy, Xiaolin Sun's cultural preservation). Demonstrates Insta360's value in a highly credible, hospital-level institutional context at one of the world's most prestigious medical institutions.

## Protocol Issues & Learnings

### Week A Search Protocol — Critical Limitation Identified

Week A specifies Reddit + YouTube searches using `site:reddit.com` and `site:youtube.com` operators. **Both are ineffective via web search:**

- Reddit posts do not surface through Google's site: operator in this tool environment
- YouTube individual video pages do not surface through web search
- 0 valid Reddit results returned across 5 different site:reddit.com queries
- 0 valid YouTube results returned across 4 different site:youtube.com queries

**Recommendation for future Week A runs:** Access Reddit directly via `old.reddit.com/r/insta360.json` API endpoint. For YouTube, use YouTube Data API v3 search with keyword filters. Both require direct API access rather than Google search.

### Source Blocking (403 Pattern)

The majority of valuable sources (academic journals, specialist photography publications, news sites) return HTTP 403 on WebFetch due to bot-detection. This is not a paywall — these are open-access pages accessible to humans. Stories from these sources are valid but must be flagged for human verification of the source quote.

Newly blocked sources confirmed this run: frontiersin.org, pmc.ncbi.nlm.nih.gov, practicalradonc.org, isprs-archives.copernicus.org

### Academic Healthcare Pattern

This run continues a pattern emerging from Weeks B, C, and A: **peer-reviewed clinical/healthcare studies are the highest-yield category** for finding new Insta360 stories that Insta360 itself has not yet discovered. Two RCT-confirmed healthcare stories now in the database:
- Olga Kukharuk — VR therapy for Ukrainian veterans (European Journal of Psychotraumatology)
- Joseph B. Schulz — virtual tour for radiation oncology patients (Frontiers in Oncology)
